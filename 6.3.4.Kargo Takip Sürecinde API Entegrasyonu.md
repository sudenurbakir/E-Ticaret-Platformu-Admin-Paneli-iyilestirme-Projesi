6.3.4.Kargo Takip Sürecinde API Entegrasyonu

Proje kapsamında kargo firmalarıyla API entegrasyonu yapılarak kargo süreçlerinin otomasyonu hedeflenmektedir. Bu entegrasyon sayesinde manuel kargo girişi, takip numarası oluşturma ve durum güncellemeleri büyük ölçüde otomatik hale getirilecektir.

API Entegrasyonunun Çalışma Prensibi:

Sipariş onay aşamasında sistem, ilgili kargo firmasının API servisine istek göndererek kargo kaydı oluşturacaktır.

Gönderilecek veriler arasında alıcı adı-soyadı, teslimat adresi, telefon numarası, sipariş içeriği, paket ağırlığı ve boyut bilgileri yer alacaktır.

Kargo firmasının API’si, bu verileri işleyerek benzersiz bir kargo takip numarası üretecek ve kargo etiketini (shipping label) sistemimize döndürecektir.

Oluşan takip numarası otomatik olarak sipariş kaydına işlenecektir.

Kargo durumu değiştiğinde (Hazırlanıyor → Kargoya Verildi → Yolda → Teslim Edildi → İade Yolunda vb.) kargo firması tarafındaki sistem, webhook veya periyodik sorgulama yöntemiyle bizim sistemimize bilgi gönderecektir.

Gelen durum güncellemesi admin panelinde anlık olarak yansıtılacak, ilgili sorumlu kişilere bildirim gönderilecek ve müşteriye otomatik SMS/e-posta ile bilgi verilecektir.

Teknik Beklentiler:

Entegrasyon RESTful API yapısı üzerinden gerçekleştirilecektir.

Güvenlik için API anahtarları (API Keys), OAuth veya token-based authentication yöntemleri kullanılacaktır.

Tüm entegrasyon trafiği loglanacak ve olası hatalara karşı retry mekanizması (yeniden deneme) devreye girecektir.

Kargo firmasının API’sinde oluşabilecek kesinti durumları için manuel yedek süreç tasarlanacaktır.

Sağlanacak Kazanımlar:

Kargo oluşturma süresinin dakikalardan saniyelere inmesi

İnsan hatasının minimuma indirilmesi

Gerçek zamanlı kargo takibi

Müşteri bilgilendirme kalitesinin ciddi oranda artması

Kargo performansı raporlamasının otomatikleşmesi
